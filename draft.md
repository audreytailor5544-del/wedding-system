# 고객 예약현황 캘린더 — 서비스 초안

## 한 줄 정의
고객 예약현황 캘린더

## 사용자
- 대상: 팀 직원 (부서별)
- 사용 방식: 각 부서가 캘린더 대시보드로 예약 현황 공유

## 카테고리 구성
| 카테고리 | 설명 |
|----------|------|
| 드레스피팅 | 드레스 담당 직원 |
| 헤어메이크업 | 헤어·메이크업 담당 직원 |
| 스타일리스트 1~6 | 각각 다른 직원 (총 6명) |

- 총 8개 카테고리, 각 카테고리별 독립 캘린더로 예약 관리
- 고객 1명이 여러 카테고리를 동시에 예약 가능 (예: 드레스피팅 + 스타일리스트 2)
- 카테고리별 날짜·시간은 각각 독립적으로 설정

## 현재 방식 (AS-IS)
1. 고객이 카카오톡으로 예약 문의
2. 채팅으로 양식 작성 받음
3. timetree 앱에 수동으로 예약 내용 입력
4. 담당자를 텍스트로 수동 배정

## 불편한 점
- 계약서 내용을 앱에 일일이 타이핑해야 함
- 예약 등록, 담당자 배정이 모두 수동
- 반복 작업으로 인한 시간 낭비 및 실수 가능성

## 목표 (TO-BE)
- 계약서 내용이 캘린더에 자동 등록
- 담당자 배정을 버튼 클릭 한 번으로 처리 (내부 직원만 확인 가능, 고객에게 비공개)
- 예약 확정 시 고객에게 카카오톡 메시지 자동 발송

## 핵심 기능 (예상)
1. 캘린더 대시보드 (부서별)
2. 부서별 카테고리 예약 관리 (부서마다 담당하는 카테고리가 다름)
3. 예약 등록 (계약서 내용 자동 입력)
4. 담당자 버튼 배정 (내부 전용)
5. 카카오톡 자동 발송
