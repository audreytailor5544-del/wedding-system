# service-prd.md

## 한 줄 정의
AUDREYTAILOR 상담팀이 고객 예약을 등록하면, 전 직원이 부서별 캘린더에서 자기 일정을 확인하고 상태를 관리하는 내부 예약 관리 시스템

---

## 데이터 아키텍처
| 역할 | 도구 |
|------|------|
| 일정 시각화 | Google Calendar (카테고리별 캘린더 9개) |
| 고객 정보·예약 데이터 | Google Sheets (예약 마스터 시트) |
| 의상 사진 | 앱 대시보드 내 표시 (Vercel Blob 저장, Google Drive 연동 없음) |
| 회계 관리 | Google Sheets (회계 전용 탭) |

## UX 레퍼런스
- 대시보드·카테고리 구조: **TimeTree** 스타일
  - 카테고리별 색상 구분
  - 캘린더 뷰에서 예약 블록 클릭 → 상세 정보 표시
  - 카테고리 탭 전환으로 부서별 일정 필터

---

## 예약 양식 필드 (전체 공통)
1. 신부 성함 / 연락처
2. 신랑 성함 / 연락처
3. 의상피팅 날짜·시간
4. 스냅촬영 날짜·시간
5. 포토업체명
6. 오드리 이용상품
7. 헬퍼 서비스 신청 여부 (일반 25만 / 베테랑 50만 / 프리미엄 50만)
8. 생화부케 추가 여부 (신청 시 업체명)
9. 예약 유입경로
10. 기타 요청사항
11. 현금영수증 발행 번호
12. 선택 의상 사진 (다중 업로드 — 앱 대시보드에 표시, Vercel Blob 저장)

## 예약 상태 흐름
대기 → 확정 → 완료

## 캘린더 카테고리 (총 9개)
드레스팀 / 헤어메이크업팀 / 스타일리스트 1~6 / 회계

---

## Phase 1 — 예약 입력 + 화면 표시 (MVP)

- **사용자 가치:** 예약 양식을 입력하면 캘린더 뷰에 즉시 표시된다
- **핵심 동작:**
  상담팀이 예약 양식(11개 항목) 입력
  → 의상피팅·스냅촬영 날짜 기준으로 캘린더에 예약 블록 표시
  → 월간 캘린더 뷰에서 전체 예약 한눈에 확인
  (브라우저 상태 저장 — 새로고침 시 초기화되어도 됨)
- **화면:** 예약 입력 폼 + 캘린더 뷰 (2개)
- **외부 의존성:** 없음 (API 연동 없이 작동)
- **성공 기준:** 예약 양식을 작성하면 해당 날짜 캘린더에 예약 블록이 표시된다
- **왜 Phase 1인가:** 입력 → 시각화가 없으면 도구 자체가 성립하지 않음. timetree를 대체하는 최소 장면

---

## Phase 2 — Google Sheets 연동 (고객 정보 저장)

- **사용자 가치:** 새로고침해도 예약이 사라지지 않고, Google Sheets에서도 바로 확인할 수 있다
- **변경 내용:**
  - Google Sheets API 연동
  - 예약 입력 시 Sheets 마스터 시트에 자동 기록 (12개 필드 + 상태 + 등록일시)
  - 의상 사진 업로드 → Vercel Blob 저장 → 앱 대시보드 예약 상세에 표시
  - Sheets에서 직접 수정해도 앱에 반영
- **왜 이 순서인가:** 데이터가 저장돼야 실무에서 쓸 수 있는 도구가 됨

---

## Phase 3 — Google Calendar 연동 (일정 시각화)

- **사용자 가치:** Sheets에 저장된 예약이 Google Calendar에도 자동으로 등록된다
- **변경 내용:**
  - Google Calendar API 연동
  - 예약 등록 시 카테고리에 맞는 Google Calendar에 이벤트 자동 생성
    (드레스팀 캘린더, 헤어메이크업팀 캘린더, 스타일리스트 1~6 캘린더 각각)
  - 직원 스마트폰 Google Calendar 앱으로도 일정 확인 가능
- **왜 이 순서인가:** Sheets에 데이터가 쌓인 후 Calendar 연동이 의미 있음

---

## Phase 4 — 예약 상태 + 담당자 배정

- **사용자 가치:** 예약 상태와 담당자를 버튼 한 번으로 바꿀 수 있다
- **변경 내용:**
  - 예약 카드에 상태 버튼 추가 (대기 / 확정 / 완료)
  - 담당자 배정 버튼 UI (텍스트 입력 없이 직원 목록에서 선택)
  - 상태·담당자 변경 시 Sheets + Calendar 동시 업데이트
  - 담당자 정보는 내부 전용, 고객에게 비공개
- **왜 이 순서인가:** Sheets + Calendar 연동 완성 후 상태 변경이 실제 효력을 가짐

---

## Phase 5 — 로그인 + 부서별 권한

- **사용자 가치:** 내 부서 캘린더만 보이고, 권한 없는 페이지는 접근이 막힌다
- **변경 내용:**
  - 직원 계정 로그인 (/login → 소개 페이지 CTA 연결)
  - 부서별 접근 권한 분리
    - 상담팀: 전체 예약 등록·조회·수정 가능
    - 드레스팀 / 헤어메이크업팀 / 스타일리스트 1~6: 해당 카테고리 캘린더만 접근
    - 회계팀: 회계 탭 + 전체 예약 조회 가능
- **왜 이 순서인가:** 핵심 워크플로우 완성 후 권한 분리가 자연스러움

---

## Phase 6 — 카카오톡 자동 발송

- **사용자 가치:** 예약이 확정되면 고객에게 카카오톡이 자동으로 전송된다
- **변경 내용:**
  - 카카오톡 비즈니스 API 연동
  - 상태가 '대기 → 확정'으로 변경되는 순간 신부 연락처로 확정 메시지 자동 발송
  - 발송 내용: 예약 확정 안내 + 날짜·시간·이용 상품 요약
- **왜 이 순서인가:** 로그인·권한이 완성된 후 외부 API 연동이 안전하고 책임 추적 가능

---

## Phase 7 — 회계 관리

- **사용자 가치:** 예약별 결제 정보와 정산 현황을 Google Sheets 회계 탭에서 바로 관리한다
- **변경 내용:**
  - Google Sheets 회계 전용 탭 연동
  - 예약 확정 시 회계 탭에 자동 행 추가 (이용상품, 헬퍼 서비스, 부케 등 금액 항목)
  - 회계팀 전용 뷰에서 정산 현황 확인
- **왜 이 순서인가:** 예약 데이터가 안정화된 후 회계 연동이 정확함

---

## Phase 8 — 협업 메모 (댓글)

- **사용자 가치:** 예약 건마다 직원들이 변동 사항을 댓글로 기록하고 공유한다
- **변경 내용:**
  - 예약 상세 화면에 댓글 입력·조회 기능 추가
  - 작성자(직원명) + 작성 시각 + 내용 표시
  - 모든 직원 댓글 작성 가능, 삭제는 본인만
  - Google Sheets 메모 컬럼 또는 별도 탭에 기록
- **왜 이 순서인가:** 핵심 워크플로우 완성 후 협업 기능을 얹음

---

## Phase 9 — 대시보드 (월별 예약 현황)

- **사용자 가치:** 이번 달 예약 현황을 숫자와 시각화로 한눈에 파악한다
- **변경 내용:**
  - Google Sheets 데이터 기반 월별 통계 대시보드
  - 예약 건수·상태 분포·카테고리별 현황·유입경로 요약
  - 전 직원 공통 접근
- **왜 이 순서인가:** 데이터가 충분히 쌓인 후에야 통계가 의미 있음

---
